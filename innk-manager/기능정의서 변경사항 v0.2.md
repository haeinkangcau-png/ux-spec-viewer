# innk Manager 기능정의서 변경사항 — v0.1 → v0.2

> 이전 버전: `v0.1`
>
> 현재 버전: `v0.2`
>
> 기준 문서: `기능정의서_innk-Manager_v.0.1_20260422.md` → `기능정의서_innk-Manager_v0.2.md`
>
> 작성일: 2026-04-24

---

## 1. Admin 계정 생성으로 섹션 제목 변경

섹션 제목이 "Installer 설정 단계"에서 "Admin 계정 생성"으로 변경되었습니다.

**관련 섹션**: `2.2`

---

## 2. 비밀번호 설정 화면 삭제 및 내용 재배치

독립 섹션으로 존재하던 비밀번호 설정 화면(v0.1의 2.5)이 삭제되었으며, 비밀번호 정책 내용은 9.6 개인 설정 > 패스워드 관리로 이동되었습니다. 이에 따라 기존 2.6 세션 관리는 2.5로, 2.7 기기 연결 상태 접근 권한은 2.6으로 번호가 조정되었습니다.

**관련 섹션**: `2.5`, `2.6`, `9.6`

---

## 3. 로그인 화면에 비밀번호 변경 권고 안내 추가

로그인 시 비밀번호 변경을 권고하는 동작이 새롭게 명시되었습니다. 해당 내용의 도입 여부는 확인이 필요합니다.

**관련 섹션**: `2.3`

---

## 4. 디바이스 목록 표시 항목 및 사용 현황 추가

디바이스 목록에 표시되는 항목(기기명, 자극모드, 연결 상태, 사용 현황)이 명시되었습니다. 사용 현황(대기 중 / 사용 중) 개념이 새롭게 추가되었으며, 미반납 상태에 대한 확인 필요 사항도 함께 기재되었습니다.

**관련 섹션**: `3.1`

---

## 5. 디바이스 상세 표시 항목 확장

기기 상세 화면에서 SW 버전 항목이 디바이스 버전과 펌웨어 버전으로 분리되었습니다. 사용 현황, 최근 설정일 항목이 새롭게 추가되었으며, 최근 사용 환자·모듈 상태·반납 예정일에 대한 검토 사항도 추가되었습니다.

**관련 섹션**: `3.2`

---

## 6. 디바이스 설정 제어 기능 신규 추가

디바이스 관리 화면에 기기 제어 관련 기능이 신규 서브섹션으로 추가되었습니다. LED 밝기 조절, 모듈 볼륨 조절, 모듈 상태 관리(정상·분실·고장), 시간 동기화 기능이 포함되었습니다.

**관련 섹션**: `3.3`

---

## 7. 처방 진입 시 '사용 중' 디바이스 확인 정책 추가

파라미터 설정 단계 진입 시, 현재 디바이스의 사용 현황이 '사용 중'인 경우 confirm dialog를 통해 사용자에게 알리고 확인 후 덮어쓰기를 진행하는 정책이 새롭게 정의되었습니다.

**관련 섹션**: `4.1`

---

## 8. Preset 설정 내용 설정 메뉴로 이동

v0.1에서 7.2에 직접 기술되어 있던 Preset 생성·수정·삭제 관련 상세 내용이 9.3 자극 프리셋으로 이동되었습니다. 7.2는 9.3을 참조하는 형태로 간소화되었습니다.

**관련 섹션**: `7.2`, `9.3`

---

## 9. 기록 조회 탭 명칭 변경

로그 조회 화면의 첫 번째 탭 명칭이 "모듈 로그 화면"에서 "기기 로그 화면"으로 변경되었습니다. 참조 링크명도 기기로그정의서로 함께 변경되었습니다.

**관련 섹션**: `8.1`

---

## 10. 설정 메뉴 전면 신설

v0.1에서 존재하지 않던 설정(Settings) 메뉴가 전면 신설되었습니다. 소프트웨어 정보(9.1), 사용자 관리(9.2), 자극 프리셋(9.3), 보안 설정(9.4), 디스플레이(9.5), 개인 설정(9.6)의 6개 서브섹션으로 구성되었습니다.

**관련 섹션**: `9` (신규)

---

## 11. 생산 관리 서비스 독립 앱 분리 및 관리자 설정 기능 목록 정의

생산 관리 서비스가 innk Manager 내부 기능에서 별도 service app으로 분리되었습니다. 섹션 번호가 9에서 10으로 변경되었으며, 관리자 전용 설정 기능 목록(제조번호 변경, Boot mode 설정, 로그 초기화 등)이 처음으로 정의되었습니다.

**관련 섹션**: `10`, `10.1` (신규)
