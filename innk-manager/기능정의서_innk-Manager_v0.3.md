# 1. 시스템 개요

## 1.1 디바이스 모델 및 지원 자극 유형

### 디바이스 인식

- 시스템은 USB로 연결된 기기의 모델명을 자동으로 읽어 지원 기능을 결정한다.
- 운영 모드는 디바이스 종류에 따라 고정되며 앱 내에서 전환할 수 없다.

### **지원 디바이스 및 운영 모드**

| 디바이스 | 운영 모드 | 비고 |
| --- | --- | --- |
| innk01-DW-Gen2 | 임상 (Clinical) 고정 | — |
| innk02-Gen2 | 임상 (Clinical) 고정 | — |
| RUO Device | 연구 (Research) 고정 | — |

### **디바이스별 지원 자극 유형**

| 자극 유형 | innk01-DW-Gen2 | innk02-Gen2 | RUO Device |

| tACS | ✓ | — | ✓ (Research 범위) |

| tRNS | — | — | ✓ |
| AM-tACS | — | — | ✓ |

## 1.2 운영 모드 (Operation Mode)

### 운영 모드

운영 모드는 연결된 디바이스 종류에 따라 자동으로 결정되며, 앱 내 수동 전환은 불가하다.

현재 활성 운영 모드 는 RUO Device 연결 시에만 화면에 표시된다.

### **디바이스별 운영 모드**

|  | 임상 디바이스(Clinical) | 연구용 디바이스(Research) |
| --- | --- | --- |
| 적용 디바이스 | innk01-DW-Gen2, innk02-Gen2 | RUO Device |
| 허용 자극 유형 | innk01-DW-Gen2: tDCS·tACS / innk02-Gen2: tDCS | tDCS·tACS·Sham·tPCS·tRNS·AM-tACS (전체 6종) |
| 파라미터 범위 | 사전 정의된 임상 기준 범위로 제한 | 임상 모드 대비 확장된 범위 허용 |
| 1일 세션 횟수 | 1회 고정 | 1\~5회 |
| 비고 | Sham모드는 gen2에서는 제외됨 |  |

# 2. 인증 및 접근 제어

## 2.1 사용자 권한 체계

Admin과 일반 사용자 두 가지 권한을 지원하며, 권한에 따라 접근 가능한 기능이 구분된다.

| 메뉴 | Admin | 일반사용자 |
| --- | --- | --- |
| 디바이스 | ✓ | ✓ |
| 자극설정 | ✓ | ✓ |
| 로그 조회 &gt; 모듈 로그 | ✓ | ✓ |
| 로그 조회 &gt; Manager 로그 | ✓ | — |
| 계정 및 설정 &gt; 개인 설정, 정보 조회 | ✓ | ✓ |
| 계정 및 설정 &gt; 전체 메뉴 | ✓ | — |

# 3. 디바이스 관리

## 3.1 디바이스 연결

### 디바이스 목록 영역

- USB로 연결된 디바이스 상태를 실시간으로 감지하고 표시한다.
- 연결 상태 변경 시 자동으로 목록을 갱신한다.
- 디바이스 선택 시 상세 정보가 표시된다.

\[확인 필요\] 디바이스에 따른 운영 모드 전환 시 사용자 안내 필요 여부 검토

### 제공 정보

| 명칭 | 설명 | 비고 |
| --- | --- | --- |
| 기기명 | 등록된 기기의 이름 |  |
| 자극모드 | 해당 기기에 설정된 자극 모드 |  |
| 연결 상태 | 하단 연결 상태 상세 참고 |  |
| 사용 현황 | 사용 현황 상세 참고 | (ybrain) |

**연결 상태 상세**

| 연결 상태 | 설명 | 비고 |
| --- | --- | --- |
| Manager 연결 | PC에 연결되어 있고, innk Manager와도 연결된 상태 | 1대만 가능 |
| PC 연결 | PC에 연결되어 있으나 innk Manager와는 연결되지 않은 상태 | 여러 대 가능 |
| 미연결 | 현재 연결되지 않았으나 과거 연결 기록이 있는 상태 | 여러 대 가능 |

**사용 현황 상세**

- 대기 중 : 자극 설정이 입력되지 않은 디바이스
- 사용 중: 자극 설정이 입력된 디바이스

\[확인 필요\] ybrain에서는 원외처방으로 반출되었으나 반납 기한을 넘긴 모듈에 대한 '사용 현황 - 미반납'상태도 관리하고 있음

### **디바이스 목록 내 제공 기능**

| 기능 | 설명 |
| --- | --- |
| 새로 고침 | 기기 목록을 새로 고침한다 |
| 기기 연결 | 선택한 기기와 innk Manager를 연결한다 |

기기가 연결되지 않은 경우 메뉴별 접근/미접근 관리 정책은 2.6 기기 연결 상태에 따른 접근 권한 참조

# 4. 원내처방

**원내 처방 : 파라미터 설정** → **설정 정보 확인** → **환자 식별 코드 입력** → **전송**

## 4.1 파라미터 설정

### 설정 단계 진입

- 설정 단계 진입 시, 현재 디바이스의 '사용 현황' 이 '사용 중'인 경우 confirm dialog를 노출하여 본 디바이스가 이미 설정된 디바이스임을 알림
- 사용자의 confirm후 해당 디바이스의 설정을 덮어써서 진행

### **파라미터 불러오기**

- 기본값, Preset 또는 직접 입력을 통해 파라미터를 설정할 수 있다.
- 입력된 값은 수정 가능하다.

| 방법 | 설명 |
| --- | --- |
| 기본값 | 페이지에 최초 접근 시, 시스템에 정의된 기본값이 노출된다. |
| Preset | 저장된 Preset을 선택하여 파라미터를 불러온다. Preset의 저장·편집은 GNB &gt; 계정 드롭다운 &gt; Settings &gt; Preset 메뉴에서만 가능하며, 본 화면에서는 불가하다. |
| 직접 입력 | 기본값 또는 Preset으로 불러온 값을 원하는 값으로 조절한다. 값의 범위 및 조절 단위는 파라미터 정의서를 따른다. |

### **파형 미리보기**

- tDCS 및 tACS 처방 시 입력된 파라미터에 맞춰 파형 미리보기 버튼을 제공한다.
- 버튼 선택 시 별도의 Modal을 통해 현재 설정된 파형을 그래프 형태로 제공한다.

### 설정 정보 확인

- 파라미터 설정 완료 후 사용자의 사용자 동작에 의해 설정 정보 확인 단계로 이동한다.

## 4.3 환자 식별 코드 입력

### **환자 식별 코드**

- 처방 전송 직전에 환자 식별 코드를 입력한다.

- 원내처방에서 환자 식별 코드는 선택 입력 항목으로 관리된다.

- 환자 식별 코드 입력 및 출력에 별도의 제한 사항은 없다.

## 4.4 자극 설정 전송

### 자극 설정 전송

- 사용자의 사용자 동작에 따라 설정한 자극 파라미터와 환자 식별 코드를 innk 디바이스로 전송한다.
- 전송 완료 시 디바이스에 자극 정보가 기록되며, 자극 수행 가능 상태로 전환된다.
- 전송 완료, 실패, 취소 시 innk Manager의 사용 로그에 기록된다.

### **전송 진행 상황**

- 별도의 전송 진행을 알리는 화면은 제공하지 않는다.

\[확인 필요\] 전송 과정의 진행 상태를 제공할 정도의 시간이 소요되는 과정인가?

### **전송 완료 화면**

- 전송이 성공적으로 완료되었음을 사용자에게 안내하는 메시지를 제공한다.
  - 완료 시 환자 ID 및 자극 유형 정보를 사용자에게 제공한다.
- 전송이 실패한 경우 오류 안내 메시지를 제공한다.

\[확인 필요\] 전송이 실패할 수 있는 상세 case를 식별하여 오류 메시지 작성 필요

## 5.2 스케줄 설정

### 기능 상세

원외처방 전용 스케줄 항목을 매 처방 시 직접 입력한다.

| 항목 | 설명 | 제약 |
| --- | --- | --- |
| 시작일 (Start Date) | 처방 시작 날짜 | 종료일 이전 또는 동일 |
| 종료일 (End Date) | 처방 종료 날짜 | 시작일 이후 또는 동일 |
| 요일 선택 | 자극을 수행할 요일 (일\~토, 다중 선택) | 최소 1개 선택 |
| 1일 세션 횟수 | 하루에 수행할 세션 수 | 임상 모드: 1회 고정 / 연구 모드: 1\~5회 |
| 자극 후 휴식 시간 | 자극 후 기기 작동이 불가한 시간 (시간 단위) | 1\~8시간, 기본값 4 |

- 환자 식별 코드 입력 및 출력에 별도의 제한 사항은 없다.
