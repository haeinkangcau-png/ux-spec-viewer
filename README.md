# UX Spec Viewer

기능정의서 MD 파일을 브라우저에서 바로 열어볼 수 있는 정적 뷰어입니다.  
GitHub Pages에 배포해서 팀이 함께 사용할 수 있습니다.

## 🚀 GitHub Pages 배포 방법

1. 이 `Spec viewer` 폴더를 **새 GitHub 레포지토리**로 푸시합니다.
2. 레포 Settings → **Pages** → Source: **GitHub Actions** 선택
3. `main` 브랜치에 push하면 자동 배포됩니다.
4. 배포 URL: `https://<your-org>.github.io/<repo-name>/`

## 📂 사용 방법

### 1. 로컬 파일 열기
- 뷰어 페이지에서 MD 파일을 **드래그&드롭** 하거나 **파일 선택** 버튼 클릭
- 파일은 브라우저 내에서만 처리되며 외부로 전송되지 않습니다

### 2. URL로 바로 열기
뷰어 URL에 `?url=` 파라미터를 붙이면 원격 MD 파일을 자동 로드합니다.

```
https://<your-org>.github.io/<repo-name>/?url=https://raw.githubusercontent.com/user/repo/main/spec.md
```

> **참고**: GitHub raw URL은 CORS 정책상 직접 fetch가 가능합니다.  
> 다른 도메인의 파일은 CORS 허용 여부에 따라 다를 수 있습니다.

### 3. MD 파일 형식

기존 기능정의서 MD 형식을 그대로 사용합니다:

```markdown
# 제목

> 버전: v1.0
> 작성일: 2026-04-22
> 작성자: 홍길동

---

# 00. 배포이력

| 버전 | 일시 | 담당자 | 내용 |
| --- | --- | --- | --- |
| v1.0 | 2026-04-22 | 홍길동 | 최초 작성 |

---

# 1. 섹션 이름

## 1.1 기능 항목 이름

### 기능 상세

기능 설명을 여기에 작성합니다.

### 예외처리

| 조건 | 처리 |
| --- | --- |
| 오류 발생 시 | 오류 메시지 표시 |
```

## 기능

- 📄 로컬 MD 파일 드래그&드롭 / 파일 선택
- 🔗 URL 파라미터로 원격 MD 자동 로드 (`?url=...`)
- 🗂️ 섹션별 사이드바 목차 네비게이션
- 🔍 섹션 클릭 시 상세 패널 표시
- 📊 배포이력 테이블 자동 파싱
- 🔗 링크관리 (`# 99.`) → `@@파일명` 링크 카드 렌더링
- 📥 MD 내보내기 / 목차 MD 내보내기
- ↔️ 상세 패널 너비 드래그 조정
- 🔤 상세 패널 글자 크기 조정 (A+ / A−)
